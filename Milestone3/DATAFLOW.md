## Where Data Enters the System

Data enters the Online Exam Management System through two roles:

**1. Teacher (Admin)**
- A teacher registers and logs in using their credentials (name, email, password).
- The teacher creates an exam by providing: title, duration, total marks, and date.
- For each exam, the teacher adds questions specifying: question text, question type (MCQ / Short Answer / True-False / Descriptive), marks per question, and the correct answer.
- All of this is written into the `Teacher`, `Exam`, and `Question` tables.

**2. Student**
- A student registers with their name, email, password, and program.
- The student logs in and sees available exams.
- When the student attempts an exam, their score is computed automatically by comparing submitted answers against `correct_answer` values in the `Question` table.
- A result record is then created and stored in the `Result` table.

## How Data Moves Through the Database

```
[Teacher registers] ──────────────────────► Teacher Table
                                                  │
                                                  │ teacher_id (FK)
                                                  ▼
[Teacher creates exam] ───────────────────► Exam Table
                                                  │
                                                  │ exam_id (FK)
                                                  ▼
[Teacher adds questions] ─────────────────► Question Table
                                                  │
                                                  │ (used during exam attempt)
                                                  ▼
[Student registers] ──────────────────────► Student Table
                                                  │
                                                  │ student_id (FK)
                                                  ▼
[Student attempts exam] ──────────────────► Result Table ◄─── exam_id (FK from Exam)
```

**Table dependency order (data must be inserted in this sequence):**

1. `Teacher` — must exist before exams can be created
2. `Student` — must exist before results can be recorded
3. `Exam` — depends on `Teacher` (references `teacher_id`)
4. `Question` — depends on `Exam` (references `exam_id`)
5. `Result` — depends on both `Student` (references `student_id`) and `Exam` (references `exam_id`)

---

## What Comes Out

The system produces the following outputs from the database:

| Output | Query Type | Tables Involved |
|--------|-----------|-----------------|
| Student's result for a specific exam | SELECT + JOIN | Result, Student, Exam |
| All exams created by a specific teacher | SELECT + JOIN | Exam, Teacher |
| All questions in a specific exam | SELECT | Question |
| Pass/Fail report for all students | SELECT + GROUP BY | Result, Student |
| Average score per exam | SELECT + AVG() | Result, Exam |
| List of students who failed | SELECT + WHERE | Result, Student |
| Total number of exams attempted per student | SELECT + COUNT() | Result, Student |

---

## Data Validation Rules

- A student cannot have two Result records for the same exam (enforced by unique constraint on `student_id + exam_id`).
- Score cannot exceed the `total_marks` defined in the Exam table.
- Every Question must belong to a valid Exam (`exam_id` FK constraint).
- Every Result must reference a valid Student and a valid Exam (FK constraints with NOT NULL).

---

## Summary

Data flows linearly from registration → exam creation → exam attempt → result generation. The `Result` table is the final destination and acts as the junction between Student and Exam, capturing each attempt's outcome. All outputs — reports, grade displays, performance summaries — are generated by querying the Result table joined with Student and Exam.
